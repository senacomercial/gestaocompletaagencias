## Story
**As a** gestor,
**I want** um dashboard dedicado ao serviço FotoIA com métricas e gestão de pedidos,
**so that** eu monitore o desempenho e gerencie o pipeline automatizado.

## Acceptance Criteria
1. `/foto-ia` — Dashboard com KPIs: leads no pipeline, taxa de conversão, receita do mês, pedidos em produção, tempo médio de entrega, taxa de aprovação 1ª rodada
2. `/foto-ia/pedidos` — Lista de pedidos com filtros e kanban visual
3. `/foto-ia/pedidos/[id]` — Detalhe: timeline do pedido, imagens geradas, histórico de execuções
4. `/foto-ia/galeria/[token]` — Página pública (sem auth) para o cliente aprovar fotos
5. Design dark Agency Gold consistente com o restante da plataforma
6. Real-time: contadores atualizam via Socket.io quando pedidos mudam de status

## Dev Notes
**Depende de:** Story 8.1 (models + API), Story 8.2-8.4 (fluxo)

**Rota galeria pública:** Deve ser acessível sem autenticação! Usar `src/app/galeria/` fora do grupo `(dashboard)`.

**Token temporário:**
```ts
// Gerar: jwt.sign({ pedidoId }, JWT_SECRET, { expiresIn: '7d' })
// Verificar: jwt.verify(token, JWT_SECRET) → { pedidoId }
```

**KPIs API:**
```ts
// GET /api/foto-ia/dashboard
{
  leadsNoPipeline: COUNT where status != ENTREGUE and != CANCELADO and != PERDIDO,
  taxaConversao: (PAGAMENTO_CONFIRMADO+ / total) * 100,
  receitaMes: SUM(valorCobrado) where entregueEm in current month,
  emProducao: COUNT where status = EM_PRODUCAO,
  tempoMedioEntrega: AVG(entregueEm - criadoEm) in ms,
  taxaAprovacao1aRodada: COUNT(aprovadoEm with rodadasRevisao=0) / COUNT(ENTREGUE) * 100
}
```
