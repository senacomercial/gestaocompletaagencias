## Story
**As a** usuário,
**I want** fazer login com email e senha,
**so that** eu acesse apenas os dados da minha organização com as permissões do meu perfil.

## Acceptance Criteria
1. NextAuth.js configurado com credentials provider (email + senha com bcrypt)
2. Session JWT contém: `userId`, `organizacaoId`, `role`, `nome`
3. Página de login em `/login` com design dark Agency Gold
4. Middleware protege todas as rotas exceto `/login`
5. Logout funcional — limpa sessão e redireciona para `/login`
6. Erro de credenciais inválidas exibido no formulário
7. Após login, redireciona para `/dashboard`

## Dev Notes
**Dependências:** `next-auth`, `bcryptjs`

**NextAuth config:**
- Provider: credentials (email + password)
- Strategy: JWT
- Secret: `NEXTAUTH_SECRET` env var
- URL: `NEXTAUTH_URL` env var

**Extensão de tipos (src/types/next-auth.d.ts):**
```ts
declare module 'next-auth' {
  interface Session {
    user: {
      id: string
      nome: string
      email: string
      role: 'ADMIN' | 'GESTOR' | 'OPERACIONAL'
      organizacaoId: string
    }
  }
}
```

**Middleware pattern:**
```ts
export { default } from 'next-auth/middleware'
export const config = { matcher: ['/((?!login|api/auth|_next).*)'] }
```
