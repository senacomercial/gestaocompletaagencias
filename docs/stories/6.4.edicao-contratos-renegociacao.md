## Story
**As a** gestor financeiro,
**I want** editar contratos e renegociar condições de pagamento,
**so that** mudanças comerciais com clientes sejam refletidas corretamente no sistema.

## Acceptance Criteria
1. "Editar Contrato": modal com campos editáveis
2. Alterar recorrência: opção "Aplicar para próximos lançamentos"
3. Renegociar: cria novo contrato, encerra anterior, cancela PENDENTES do antigo
4. Histórico de alterações do contrato
5. Encerrar contrato com motivo (CONCLUIDO, CANCELADO, INADIMPLENTE)
6. Ao encerrar: lançamentos PENDENTES → CANCELADO

## Tasks / Subtasks
- [ ] Modal de edição de contrato (AC: 1, 2)
  - [ ] Criar `src/components/financeiro/EditarContratoModal.tsx`
  - [ ] Campos editáveis: valorTotal, recorrenciaMensal, dataInicio, numeroParcelas
  - [ ] Checkbox "Aplicar para próximos lançamentos" ao alterar recorrência
  - [ ] `PUT /api/contratos/[id]`
  - [ ] Se checkbox: atualizar lançamentos PENDENTE com novo valor
- [ ] Funcionalidade de Renegociação (AC: 3)
  - [ ] Botão "Renegociar" no contrato
  - [ ] Criar `src/components/financeiro/RenegociarModal.tsx`
  - [ ] Formulário com novos termos
  - [ ] `POST /api/contratos/[id]/renegociar` — transação atômica:
    1. Criar novo `ContratoFinanceiro` com novos termos
    2. Atualizar antigo: status CANCELADO
    3. `updateMany` PENDENTE → CANCELADO nos lançamentos do antigo
    4. Criar novos lançamentos para o novo contrato
- [ ] Histórico de alterações (AC: 4)
  - [ ] Model `ContratoHistorico`: contratoId, usuarioId, campo, valorAnterior, valorNovo, createdAt
  - [ ] Registrar cada alteração nos campos do contrato
  - [ ] Seção "Histórico" expandível no detalhe do contrato
- [ ] Encerrar contrato (AC: 5, 6)
  - [ ] Botão "Encerrar Contrato" com select de motivo
  - [ ] `POST /api/contratos/[id]/encerrar`
  - [ ] Transação: status contrato → motivo, lançamentos PENDENTE → CANCELADO
  - [ ] Toast: "Contrato encerrado. [N] lançamentos cancelados."
